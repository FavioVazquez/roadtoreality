# Plan 10-03: Stop 031 — E = mc²

## Objective

Stop 031 gets the most complex single-stop sim in Phase 10. The mass-to-energy slider spans 6 orders of magnitude (0.001g to 1000g) and requires a `formatEnergy()` helper that switches units automatically. Scale references (TNT, atomic bombs, city power) are computed from verified constants and switch at defined breakpoints. A second velocity slider shows rest energy vs. relativistic kinetic energy as a stacked bar, demonstrating that E = mc² is the rest-energy special case of the full SR energy equation. Both equations appear in KaTeX with explicit narrative explaining the relationship.

## Context

This is a single-stop plan — Stop 031 alone — because it requires more implementation complexity than a two-stop plan would allow in one context window.

Key verified physics constants (from RESEARCH.md — do not re-derive at runtime):
- c = 2.998 × 10⁸ m/s
- 1 gram → E = (0.001 kg) × c² = 8.9878 × 10¹³ J ≈ 21.48 kilotons TNT
- 1 kg → E = (1 kg) × c² = 8.9878 × 10¹⁶ J ≈ 21,475 kilotons = 21.48 megatons TNT
- TNT conversion: 1 ton TNT = 4.184 × 10⁹ J
- 1 kiloton TNT = 4.184 × 10¹² J
- 1 Hiroshima-scale bomb ≈ 15 kilotons TNT
- City power (1 GW city for 1 hour) = 3.6 × 10¹² J

Critical pitfall: mass slider spans 6 orders of magnitude. Slider range must be logarithmic or the small-mass end will be unusable. Use a slider range of 0–300, map to mass in grams via: `mass_g = Math.pow(10, sliderValue / 100 - 3)` — this gives 10⁻³ g (0.001g) at slider=0 and 10⁰ kg (1000g) at slider=300. Alternatively use slider range 1–6000 mapped to grams via `Math.pow(10, sliderValue/1000 - 3)`. Either approach is valid; just be consistent.

Critical pitfall: the full energy equation is `E² = (pc)² + (mc²)²` where relativistic momentum is `p = γmv` — NOT classical `p = mv`. Compute `gamma` from beta before computing p. When v = 0, p = 0 and E = mc².

`formatEnergy(joules)` must return strings like "8.99 × 10¹³ J" at 1g and switch to human-readable comparisons at threshold breakpoints. The function should return a display string, not modify the DOM directly.

The narrative must explicitly state: "E = mc² is not the complete picture. The full relativistic energy equation is E² = (pc)² + (mc²)². E = mc² is the special case where the object is at rest (p = 0). When moving, the total energy is larger."

Established patterns:
- ES5 IIFE, DPR canvas, resize handler, `window.SimAPI` set synchronously, `drawStatic()` last statement
- Sliders declared in index.html, wired via getElementById in sim.js
- prefers-reduced-motion: drawStatic() only

   function formatEnergy(j) {

     if (j < 1e9)  return (j/1e6).toFixed(3) + ' MJ';
     if (j < 1e12) return (j/1e9).toFixed(3) + ' GJ';
     if (j < 1e15) return (j/1e12).toFixed(3) + ' TJ';
