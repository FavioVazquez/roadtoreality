# Plan 10-04: Stops 032 + 033 — Rutherford + Bohr

## Objective

Implement two simulations requiring the most animation state management in Phase 10. Stop 032 needs a particle pool for the gold-foil scattering experiment with Coulomb-correct deflection geometry and a mode toggle between stream and manual aim. Stop 033 requires a deliberate two-phase UX: classical spiral collapse (the problem) plays first for at least 3 seconds with a crash flash at the nucleus, then and only then does the Bohr toggle unlock — showing stable quantized orbits and building the Balmer emission spectrum live as electrons jump.

## Context

Stop 032 — Rutherford scattering:
- The teaser has hardcoded beam rows with fixed deflections. The full sim replaces this with a particle pool using real Coulomb deflection.
- Rutherford formula (normalized units): `cot(theta/2) = 2 * b * E_kin / (k * Z1 * Z2 * e2)`. In normalized canvas units, simplify to: `cot(theta/2) = b / d_min` where `d_min` is the closest approach distance, which scales with nuclear charge Z.
- In practice: define `D = chargeScale * 0.5` (half-canvas-height) as the "distance of closest approach for b=0 head-on collision". Then `theta = 2 * Math.atan(D / b)` for impact parameter b > 0. For b > 3*D, theta < ~37 degrees (slight deflection). For b < D, theta > 90 degrees (large deflection).
- Stream mode particle pool: maintain 8 alpha particles. Each has an assigned impact parameter sampled from a random distribution weighted toward large b (most go straight). When a particle exits the canvas right edge or bottom/top, reset it to the left edge with a new random impact parameter.
- Manual aim mode: slider sets impact parameter b from 0 (head-on) to canvas_height*0.45 (far glancing blow). Single alpha particle fires; deflection angle computed from formula; trajectory shown as two straight-line segments (incoming + outgoing) with angle label.
- Nucleus charge slider Z: range 1–79 (gold = 79). Default = 79. `D` scales linearly with Z. Higher Z = more large-angle deflections in stream mode.

Stop 033 — Bohr atom:
- Classical mode animation: use `r(t) = r_max * Math.exp(-k * (t - t_start))` with k chosen so collapse takes ~3.5 seconds (k ≈ 0.8 if t is in seconds). Electron angle advances faster as r shrinks (spiral). After r < 5 pixels: draw a white flash/burst at nucleus center; then freeze the animation and set `collapseComplete = true`.
- Toggle to Bohr mode: button has `disabled` attribute while `!collapseComplete`. When user clicks toggle: clear the canvas, draw stable circular orbits (n=1,2,3,4 with radii r_n = r1 * n²), animate electron on whichever orbit the user last selected.
- Energy levels: `E_n = -13.6 / (n * n)` eV. When electron drops from level n_upper to n_lower: photon energy `dE = 13.6 * (1/(n_lower*n_lower) - 1/(n_upper*n_upper))` eV. Wavelength `lambda_nm = 1240 / dE` (using hc ≈ 1240 eV·nm).
- Balmer series (n_lower = 2): Hα n=3→2: 656.3nm (red `rgb(255,80,60)`), Hβ n=4→2: 486.1nm (cyan `rgb(80,210,200)`), Hγ n=5→2: 434.0nm (violet `rgb(160,80,220)`), Hδ n=6→2: 410.2nm (deep violet `rgb(110,50,180)`).
- Emission spectrum bar: horizontal bar at bottom of canvas, span 380nm to 700nm. Each time an electron drops and emits a photon, draw a vertical line at the correct wavelength position on the spectrum bar. Lines accumulate (persist) — the spectrum builds over time.
- User interaction: click an orbit ring in Bohr mode to move the electron to that orbit (raising or lowering energy level). On lowering: animate a photon flying outward, add spectrum line.

Critical constraint from RESEARCH.md: Stop 032 continuous stream mode must not pile up particles near the nucleus. Use a minimum spawn interval: new particle spawns only if the previous particle from that "lane" has cleared x > canvas_width * 0.6. The pool of 8 particles handles this naturally if particles are recycled only when off-canvas.

Critical constraint from RESEARCH.md/CONTEXT.md: Stop 033 classical collapse must last at least 3 seconds and end visibly (crash flash). The Bohr toggle button must be disabled during collapse. This pacing is not optional — it is the pedagogical design.

Established patterns:
- ES5 IIFE, DPR canvas, resize handler, `window.SimAPI` set synchronously, `drawStatic()` last statement
- Sliders declared in index.html, wired via getElementById in sim.js
- prefers-reduced-motion: drawStatic() only, skip collapse animation (show both states side by side instead)

     - Set `flashAlpha = 1.0` (flash starts)
     - Enable the Bohr toggle button: `collapseBohrBtn.disabled = false;`
   - When `flashAlpha > 0`: draw white radial burst at nucleus, decrement `flashAlpha -= 0.03` each frame.

5. BOHR MODE ANIMATION (phase === 'bohr'):
   - Draw nucleus (gold dot), draw 4 orbit rings (n=1 to 4, dashed gray circles at BOHR_RADII).
   - Draw electron on current orbit ring at `bohrAngle`. Advance `bohrAngle += BOHR_SPEEDS[bohrLevel]`.
   - Draw emission spectrum bar at bottom of canvas: horizontal band from 380nm to 700nm. Background is a dim rainbow gradient. Each `spectrumLines` entry is drawn as a bright vertical line at its x_canvas position.
