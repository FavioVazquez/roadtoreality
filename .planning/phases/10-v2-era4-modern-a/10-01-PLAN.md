# Plan 10-01: Stops 027 + 028 — Planck + Photoelectric Effect

## Objective

Implement two full interactive simulations to replace stub teasers: Stop 027 shows the blackbody spectrum with a live Rayleigh-Jeans vs. Planck curve comparison controlled by temperature and Planck constant sliders; Stop 028 shows the photoelectric effect with frequency/intensity sliders, electron ejection visuals, and a live KE readout. Both stops get updated index.html narrative sections and KaTeX equations, and their stops.json entries are flipped to `isStub: false`.

## Context

All 7 Phase 10 stops currently have `isStub: true` in stops.json. Each stop already has a full HTML shell with stop-config JSON, sim-mount div, stop-skeleton, play/pause/reset buttons, search modal, and all JS/CSS includes. The existing teaser sim.js for Stop 027 already contains working `planck()` and `rayleigh()` functions using the dimensionless form `B(f) = x³/(exp(x)−1)` where `x = hf/kT` — extend this rather than starting from scratch.

Key patterns required (from RESEARCH.md and existing Phase 09 sims):
- ES5 IIFE: `(function () { 'use strict'; ... }())`
- `window.SimAPI = { start, pause, reset, destroy }` set synchronously before IIFE closes
- DPR-aware canvas: `var dpr = window.devicePixelRatio || 1; canvas.width = W * dpr; ctx.setTransform(dpr, 0, 0, dpr, 0, 0);`
- Resize handler: `window.addEventListener('resize', function() { resize(); if (!running) drawStatic(); });`
- Sliders declared in index.html, wired in sim.js via `document.getElementById`
- `prefers-reduced-motion` check: if `reduced`, call `drawStatic()` and do not start rAF loop
- `window.SimAPI.start()` is called by stop-shell.js when IntersectionObserver fires

Stop 027 critical pitfall: when h slider value approaches 0, `x = (h_factor * freq) / (T/T_ref)` approaches 0; `x³/(exp(x)−1)` must converge to `x²` (the Rayleigh-Jeans regime). The normalization factor must be the same for both curves — if they use different normalizations they will diverge instead of converge. Verify visually at h_factor = 0.01 that the Planck curve overlaps the Rayleigh-Jeans curve.

Stop 028 critical requirement: the "No effect" feedback (explicit text label on canvas or readout div showing "No ejection — frequency too low") must appear even when intensity is at maximum. This is the pedagogical point. The KE readout must show 0 eV (not a negative number rendered confusingly) when below threshold, with explanatory text.

2. STATE: `var temperature = 4000;` (Kelvin, slider range 1000–10000). `var hFactor = 1.0;` (dimensionless multiplier on h, slider range 0.01–1.0, default 1.0). These are the only two sliders.

3. SPECTRUM CURVES: Draw on a canvas area with axes. X-axis = frequency (0 to f_max, normalized). Y-axis = spectral radiance (normalized to canvas height).
   - Planck curve (solid, color matching temperature: red at 1000K, white at 6000K, blue-white at 10000K): `B_planck(x) = x³ / (Math.exp(x) - 1)` where `x = hFactor * (freq / T_norm)` and `T_norm = temperature / 4000`. Normalize peak to ~80% of canvas height.
   - Rayleigh-Jeans curve (dashed, fixed amber/orange color): `B_rj(x) = x²` using the same frequency/normalization scale as Planck. This is the classical limit.
   - When hFactor is 1.0 and T is 4000K, both curves should agree at low frequencies and diverge sharply at high frequencies (Rayleigh-Jeans blows up; Planck does not).
   - When hFactor approaches 0.01, the Planck curve should visually collapse onto the Rayleigh-Jeans curve.

4. CANVAS BACKGROUND TINTING: At low temperature (1000K), canvas background is a deep red-brown. At 6000K it is dark with blue-white tint. Use `oklch` or a simple RGB lerp between `rgb(40,8,2)` at 1000K and `rgb(10,14,28)` at 10000K.

5. LABELS: Draw text on canvas: temperature value in Kelvin (bottom-left), h factor as "h × [value]" (bottom-right), axis labels "Frequency →" (x-axis), "Radiance" (y-axis).

6. LEGEND: Draw a small legend inside the canvas: solid line = "Planck (quantum)", dashed line = "Rayleigh-Jeans (classical)".

7. ANIMATION: When running, animate a slow temperature sweep (auto-cycle 1000K → 10000K → 1000K at ~200K/second) to show Wien's law peak shift in action. When paused, show static curves at the slider values.

9. SimAPI: Set `window.SimAPI = { start: start, pause: pause, reset: reset, destroy: destroy }` synchronously.

10. `drawStatic()` is called as the last statement before the IIFE closes (so stop-shell.js skeleton-poll finds a rendered canvas immediately).

ES5 constraints: no arrow functions, no const/let, no template literals in complex expressions (simple string concatenation is fine), no destructuring.
