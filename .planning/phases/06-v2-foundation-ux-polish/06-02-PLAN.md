# Plan 06-02: KaTeX Integration

## Objective

Self-host KaTeX (the fast LaTeX math renderer) under `Episodio4/assets/katex/`, create a shared deferred init script `assets/js/katex-init.js`, add KaTeX CSS and scripts to all 12 implemented stop pages, and add a prominent display equation to the takeaway box of each stop. This delivers FEAT-KATEX-01 through FEAT-KATEX-04 and ensures every future stop gets math rendering automatically by following the same template.

## Context

**Why self-hosted:** Per DEC-002 (no CDN dependencies at runtime), all vendor files must be local. KaTeX v0.16.x is ~155 KB total (CSS + JS + fonts).

**Load order is critical (from RESEARCH.md):** Inline `<script>` tags do NOT wait for `defer`red external scripts. The `renderMathInElement` call must be in its own deferred external file. Correct load order before `</body>`:
```html
<script defer src="../../assets/katex/katex.min.js"></script>
<script defer src="../../assets/katex/auto-render.min.js"></script>
<script defer src="../../assets/js/katex-init.js"></script>
```
Deferred scripts execute in document order relative to each other, so this guarantees `katex-init.js` runs after `auto-render.min.js`.

**Font path:** KaTeX CSS references fonts with relative paths (`fonts/KaTeX_Main-Regular.woff2`). Keep the entire KaTeX distribution intact: CSS at `assets/katex/katex.min.css`, fonts at `assets/katex/fonts/`. Do NOT relocate fonts. Do NOT edit `katex.min.css`.

**Scope:** Auto-render is scoped to `.takeaway-box` first, falling back to `document.body`. This avoids false matches on dollar signs in prose text.

| Stop | Display equation | Description |
|------|-----------------|-------------|
| 001 Thales | `$$\text{Natural event} \xrightarrow{\text{natural cause}} \text{Testable prediction}$$` | The scientific method arrow |
| 003 Democritus | `$$\text{matter} = \text{atoms} + \text{void}$$` | Atomic hypothesis |
| 004 Aristotle | `$$v \propto \frac{F}{m}$$` (wrong) vs `$$a = \frac{F}{m}$$` (Newton, correct) | Aristotle's proportionality vs Newton |
| 005 Archimedes | `$$F_b = \rho_{\text{fluid}} \cdot V_{\text{displaced}} \cdot g$$` | Buoyancy force |

| 007 Ptolemy | `$$\lambda(t) = \lambda_0 + \omega_1 t + r \sin(\omega_2 t + \phi)$$` | Epicycle position formula |

<name>Create assets/js/katex-init.js</name>

<action>
Create `Episodio4/assets/js/katex-init.js` with the following exact content:

```javascript
/* ============================================================
   katex-init.js — Deferred KaTeX auto-render initializer
   Must be loaded AFTER katex.min.js and auto-render.min.js
   (all three deferred — execution order is preserved by defer)
   How Physics Works — v2.0
   ============================================================ */
(function () {
  'use strict';

  if (typeof renderMathInElement !== 'function') return;

  var container = document.querySelector('.takeaway-box') || document.body;

  renderMathInElement(container, {
    delimiters: [
      { left: '$$', right: '$$', display: true  },
      { left: '$',  right: '$',  display: false }
    ],
    throwOnError: false,
    errorColor: 'var(--color-text-muted)'
  });
}());
```

This file is intentionally minimal. It is deferred, so it runs after `auto-render.min.js` has executed. The `typeof renderMathInElement !== 'function'` guard prevents a silent failure if the load order is ever wrong. Scoping to `.takeaway-box` first avoids dollar-sign conflicts in prose text.
</action>
