# Plan 06-01: CSS & Global UX Polish

## Objective

Make three targeted CSS changes that improve reading comfort, accessibility contrast, and mobile usability across the entire site. All changes are single-token or single-rule additions with no layout side-effects. This plan intentionally excludes JS changes — those are in Plans 06-03 and 06-04.

## Context

All site CSS lives in `Episodio4/assets/css/`. The two files to modify:

- `base.css`: contains the `:root` design tokens and the `body` rule. `--color-text-muted` is currently `oklch(0.50 0.02 90)`. `body { line-height: 1.7; }` is at line 118.
- `simulation.css`: contains the `.sim-container` rule with `min-height: 360px`. The existing `@media (max-width: 640px)` block handles `.sim-controls` and `.sim-actions` but does NOT yet set `.sim-actions` buttons to 50% width or the `sim-container` mobile override.

RESEARCH note on line-height: Setting `body { line-height: 1.9; }` only affects elements that inherit without overriding. Specific overrides in `simulation.css` (`.stop-body p { line-height: 1.8; }`, `.stop-bridge { line-height: 1.7; }`, etc.) are unaffected — this is intentional per CONTEXT.md ("approach (a) — change only the base body default").

RESEARCH note on `--color-text-muted`: Changing from `0.50` to `0.55` lightens every consumer of this token. All consumers remain clearly subordinate to `--color-text-secondary` at `0.70 L`. The change is safe.

<action>
Make two additions to `Episodio4/assets/css/simulation.css`:

1. Append a new `@media (max-width: 768px)` rule after the existing `@media (max-width: 640px)` block at the end of the file:

```css
/* ── Mobile sim viewport ────────────────────────────────── */
@media (max-width: 768px) {
  .sim-container {
    min-height: 60vh;
  }
}
```

2. Inside the existing `@media (max-width: 640px)` block, add two property declarations to the existing `.sim-actions` rule so it reads:

```css
.sim-actions {
  margin-left: 0;
  width: 100%;
  justify-content: stretch;
  display: flex;
  gap: var(--space-2);
}
```

And add a new rule inside that same `@media (max-width: 640px)` block:

```css
.sim-actions .sim-btn {
  flex: 1;
  min-height: 48px;
  justify-content: center;
}
```

Note: The existing `.sim-btn { flex: 1; justify-content: center; }` rule inside the 640px block applies to ALL `.sim-btn` elements. Replace that rule with `.sim-actions .sim-btn` so the scope is explicit and the rest of the layout is unaffected. Keep all other existing rules in the 640px block unchanged.
</action>
