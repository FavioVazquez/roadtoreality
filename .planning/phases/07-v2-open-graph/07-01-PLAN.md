# Plan 07-01: OG SVG Generation

## Objective

Write `scripts/generate-og-svgs.mjs` at the repo root. The script reads `Episodio4/assets/data/stops.json` and writes one 1200x630 SVG file per stop to `Episodio4/stops/{stop.id}/og-image.svg`. Running the script produces all 50 social sharing images in a single pass. This satisfies FEAT-OG-01 and FEAT-OG-03.

## Context

The `scripts/` directory does not yet exist at repo root — create it before writing the script file.

All site code lives under `Episodio4/`. Stop directories are named by `stop.id` (e.g., `001-thales-natural-causes`), not `stop.slug`. All 50 stop directories already exist with an `index.html` inside each. None have `og-image.svg` yet.

**Era hex color map** (pre-converted from oklch — do NOT use oklch in SVG attributes, social crawlers do not support it):
- `ancient` → `#db9152`
- `revolution` → `#61bd67`
- `classical` → `#0099f0`
- `modern` → `#be6df2`
- `contemporary` → `#ff4a5d`

Dark background: `#1a1a1a` (equivalent to oklch 0.10 0 0).

**SVG text escaping:** 15 stop titles contain apostrophes (e.g., "Galileo's Pendulum") and at least one scientist name contains `&`. All text placed in SVG `<text>` elements must pass through an `escapeXml()` helper that replaces `&` with `&amp;`, `<` with `&lt;`, `>` with `&gt;`, `"` with `&quot;`, and `'` with `&#39;`.

**Typography:** Use a system-safe font stack: `font-family="Georgia, 'Times New Roman', serif"` for the title and scientist name. Web fonts do not load in crawler contexts, so do not reference the self-hosted Cormorant Garamond WOFF2 files.

**Title sizing:** The longest stop title is 38 characters ("Special Relativity: Length Contraction"). At `font-size="58"` with Georgia bold, a 38-character title fits within 1100px at `x="80"`. Use 58px for title text. No word-wrap logic is needed — all titles fit on one line at this size.

**Script behavior:** The script should print a summary line to stdout, e.g., `Generated 50 SVGs (0 errors)`. It must be idempotent: overwriting an existing SVG with the same content is acceptable.

## Known Limitation

SVG `og:image` rendering on Twitter and LinkedIn is unreliable. Twitter's card crawler does not officially support SVG; it accepts JPG, PNG, WEBP, and GIF only. Social previews may show a broken image on those platforms. This is not a bug to fix in Phase 07 — a Playwright-based PNG conversion path is already documented as a v3.0 candidate in REQUIREMENTS.md.
