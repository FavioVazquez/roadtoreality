# Plan 07-02: OG Meta Tag Injection

## Objective

Write `scripts/inject-og-meta.mjs` at the repo root. The script reads `Episodio4/assets/data/stops.json` and injects 5 Open Graph and Twitter Card meta tags into every stop's `Episodio4/stops/{stop.id}/index.html`. This satisfies FEAT-OG-02. The script is idempotency-guarded: re-running it on already-tagged files skips injection and exits cleanly.

## Context

All 50 stop pages live at `Episodio4/stops/{stop.id}/index.html`. The `stop.id` field (e.g., `001-thales-natural-causes`) is the directory name — do not use `stop.slug`.

**Head section structure (identical across all 50 files):**
```html
<meta charset="UTF-8">
<meta name="viewport" ...>
<meta name="description" content="...">
<title>...</title>
```
The `<meta name="description">` tag is always on line 6 with consistent single-line format. Injection anchor: find the `<meta name="description"...>` line and insert the 5 OG tags immediately after it.

**Idempotency guard:** Before injecting, check `if (html.includes('property="og:title"'))` — if true, skip the file and count it as already-done. Do not replace or re-inject.

**The 5 meta tags to inject (in this order, each on its own line, indented with 4 spaces to match existing head indentation):**

```html
    <meta property="og:title" content="{title} — How Physics Works">
    <meta property="og:description" content="{description}">
    <meta property="og:image" content="https://faviovazquez.github.io/roadtoreality/stops/{id}/og-image.svg">
    <meta property="og:type" content="article">
    <meta name="twitter:card" content="summary_large_image">
```

**og:image URL:** Must be `https://faviovazquez.github.io/roadtoreality/stops/{stop.id}/og-image.svg`. No `/Episodio4/` segment — the gh-pages branch serves `Episodio4/` content at the site root.

**HTML attribute escaping:** The `content` attribute values for `og:title` and `og:description` must use HTML attribute escaping. Specifically, `&` must become `&amp;` and `"` must become `&quot;` in attribute values. Apostrophes in titles (15 stops) are safe unescaped inside double-quoted attributes. A small `escapeHtmlAttr(str)` helper handles this: replace `&` → `&amp;`, then `"` → `&quot;`.

**Injection method:** Use `String.prototype.replace()` with the pattern matching the `<meta name="description"[^>]+>` line. Append the 5-tag block immediately after the matched tag on the same replacement. Do not use line-number-based injection — the head structure could change in later phases.

**og:description length:** The longest stop description in stops.json is approximately 180 characters. All descriptions fit within the 200-character recommended limit for OG descriptions without truncation.

## Known Limitation

SVG `og:image` files may not render as social previews on Twitter or LinkedIn because those platforms do not reliably support SVG as an og:image format. The meta tags are correct and complete — the limitation is the SVG format itself. A Playwright-based PNG conversion is documented as a v3.0 candidate in REQUIREMENTS.md. Do not treat missing Twitter/LinkedIn previews as a bug to fix in this phase.
