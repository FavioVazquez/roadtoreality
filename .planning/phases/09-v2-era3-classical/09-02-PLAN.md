## Context

Both stops are `isStub: true`. Reference: stop-014 sim.js (ES5 IIFE pattern). Key notes for this plan:

**Stop 017 (Coulomb) — field line tracing:** Use Euler-step integration from seed points around each positive charge. Seed 8 rays per positive charge at angles 0, 45, 90, 135, 180, 225, 270, 315 degrees, starting 12px from charge center. Each step: compute net E-field at current position (sum of contributions from all charges), normalize, multiply by step size (~3px). Termination: (a) if position exits canvas bounds, (b) if within 10px of any charge center (acts as sink), (c) max 500 steps. Same-sign repulsion: lines curve away from the like charge. Opposite-sign: lines curve toward the negative charge. This is physically correct — follow the actual E-field direction.

**Stop 017 — split-canvas layout:** Left panel = charge positions + field lines + force arrow. Right panel = F vs. r graph. `splitX = W * 0.55` (field panel gets slightly more room). Compute `splitX` inside `resize()`.

**Stop 018 (Volta) — circuit drawing:** Draw the circuit schematically. Battery = series of alternating long/short horizontal lines (standard cell symbol). Load resistor = zigzag line. Current arrows on the wires. In series mode: cells stack top to bottom; V_total = n * V_cell. In parallel mode: cells are side by side branches; V_total = V_cell (1 cell equivalent), I_total = n * (V_cell / R_load).

**ES5 constraint:** All `var`, named functions, string concatenation. No arrow functions, no const/let, no template literals.

Era color: `rgba(82,133,200,0.8)` for canvas drawing.

**Field line tracing function `traceFieldLines()`:**
- Called each frame (or each time charges move).
- For each charge with q > 0 (positive), seed N_rays=8 rays at angles i*2*PI/N_rays starting 12px from charge center.
- Euler step: at position (px, py), compute net E = sum over all charges of (k*q_i / r_i^2) * unit_vector_from_charge_to_point. Step: px += E.x/|E| * stepSize; py += E.y/|E| * stepSize. stepSize = 3.
- Termination per ray: stop if (px < 0 || px > splitX || py < 0 || py > H) for canvas bounds; stop if distance to any charge center < 10px; stop if steps > 500.
- Collect path points and draw as a polyline with `ctx.strokeStyle = 'rgba(82,133,200,0.5)'`, lineWidth = 1.
- Performance note: with 2 charges and 8 rays each = 16 traces × 500 steps max = 8000 iterations per frame. This is acceptable.

**Charge drag interaction:**
- `mousedown`/`touchstart`: find closest charge within 20px; set `dragging = true` on it.
- `mousemove`/`touchmove`: if dragging, update charge x,y. Event normalization: `var pos = e.touches ? e.touches[0] : e; var rect = canvas.getBoundingClientRect(); var cx = (pos.clientX - rect.left); var cy = (pos.clientY - rect.top);`
- `mouseup`/`touchend`: clear dragging. Use `touch-action: none` — already in simulation.css.
- Add both mouse and touch listeners in `setup()`. Do NOT use pointer events API.

**Force calculation:**
- k = 8.99e9 (Coulomb's constant). Use normalized q values (+1 or -1). For display: F = k * |q1*q2| / r^2. Normalize to a display-scale value: F_display = 1 / r_px^2 where r_px is pixel distance. Show an actual force arrow between the two charges. Arrow length = min(80, 5000/r_px^2) pixels. Direction: repulsion = arrows pointing away from each charge; attraction = arrows pointing toward each other.
- Live readout text at bottom of left panel: "F = " + F_display.toFixed(3) + " (normalized)"  and  "r = " + r_px.toFixed(0) + " px"

**Right panel — F vs. r graph:**
- Plot the curve F = C/r^2 (with C = 1 for same-sign, C = -1 for opposite shown as attractive). x-axis: r from 20px to (splitX - 50px) range. y-axis: F_normalized from 0 to max. Graph axes, labels ("F (N·m²)", "r (px)"). Live tracking dot at current (r_px, F_display) position. Same as stop-014 graph pattern.

2. `charge-magnitude-slider` (range 0.1–3.0, step 0.1, default 1.0): scales both |q| values (uniform scaling). Updates force readout.
