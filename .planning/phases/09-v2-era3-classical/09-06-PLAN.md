# Plan 09-06: Stops 025 + 026 — Hertz Radio Waves and Michelson-Morley

## Objective

Implement two full simulations closing out Era 3: (1) Hertz's oscillating dipole emitting EM radiation with sin²θ angular distribution and color-coded E-field direction; (2) the Michelson-Morley experiment — the most complex sim in the entire phase, featuring three fully interactive modes (expected vs. found, speed-of-light race, interferometer) accessible via tab switching within a single canvas. Stop 026 is a ~700-line sim.

## Context

Both stops are `isStub: true`. Reference: stop-014 sim.js (ES5 IIFE pattern). This is the final plan of Phase 09.

**Stop 025 — Hertz dipole radiation pattern:**
- sin²θ radiation pattern: intensity is zero along the dipole axis (θ=0°, 180°) and maximum perpendicular (θ=90°, 270°). Draw expanding ellipses (not circles) that are wider perpendicular to the dipole. OR draw expanding rings whose opacity at each point (angle θ from dipole axis) is proportional to sin²θ. Either approach is valid.
- E-field direction alternates: each half-cycle emits with opposite E-field polarity. Color code: red for positive half-cycle wavefronts, blue for negative half-cycle wavefronts. Use birth_phase to track which half-cycle produced each ring.

**Stop 026 — THREE-MODE critical details (RESEARCH.md + CONTEXT.md):**
- One canvas, one `draw()` function. `currentMode` variable (1, 2, or 3). `draw()` starts by clearing canvas, then calls exactly one of `drawModeExpected()`, `drawModeRace()`, or `drawModeInterferometer()` based on `currentMode`. See RESEARCH.md Pitfall 6.
- Three tab buttons (not sliders) declared in HTML: `mode-1-btn`, `mode-2-btn`, `mode-3-btn`. Clicking sets `currentMode` and updates button active state. Add `.is-active` class or inline style to highlight the active tab.
- Mode 3 interferometer: use `Math.cos(2 * Math.PI * pathDiff / lambda)` per pixel row for fringe pattern. Canvas width ~250–300px of the right third. DO NOT draw individual wavefronts — compute intensity directly per scanline. Fast enough at 400–600px canvas widths.

**ES5 constraint:** All `var`, named functions, string concatenation. No arrow functions, no const/let, no template literals.

Era color: `rgba(82,133,200,0.8)`.

**Mode 1 — Expected vs. Found (fringe shift prediction):**

Split canvas vertically at W*0.5. Left half = "Expected (Aether Hypothesis)". Right half = "Observed (Actual Result)".

Left sub-panel — predicted fringe shift:

- Draw: a sine wave at the center of the left sub-panel. `amplitude = 0.3 * Math.abs(Math.sin(rotationAngle * Math.PI / 180))` (modulates with rotation angle — max shift at 0° and 180°, min at 90° and 270°). - Draw as: center horizontal line (axis), then a sine wave above/below it. `y_i = cy + amplitude * maxAmp * Math.sin(i * 2*PI/waveWidth)` for i across the sub-panel width.
- The sine wave shifts up/down as rotation changes — illustrating that predicted fringes should change with rotation.
- Label: "Expected fringe shift" at top. "Rotation: " + rotationAngle.toFixed(0) + "°" below label. Axis label: "Fringe position".

Right sub-panel — observed (null) result:
- Draw: a flat straight line at vertical center. No oscillation, no shift, regardless of rotation.

**Mode 2 — Speed-of-light race:**

Draw the interferometer as a top-down schematic centered in the canvas.
- Beam splitter: small square at canvas center (cx = W*0.5, cy = H*0.5).
- Two arms: one horizontal (arm A, length armLen_px toward right), one vertical (arm B, length armLen_px toward top). Both the same length. armLen_px = Math.min(W, H) * 0.35.
- Arm endpoints: mirror A at (cx + armLen_px, cy), mirror B at (cx, cy - armLen_px).
- Draw the two arms as lines from center to mirror positions.
- Rotate the entire apparatus by `rotationAngle`: apply `ctx.translate(cx, cy); ctx.rotate(rotationAngle * Math.PI / 180); ctx.translate(-cx, -cy)` before drawing.
- Two animated light dots: dot_A travels along arm A (cx → mirror_A → cx), dot_B travels along arm B (cx → mirror_B → cx). Both use `racePhase` (0.0–1.0, cycles through 0→1 repeatedly). `racePhase += 0.008` per frame (can be adjusted). Both dots advance at the SAME rate — they ALWAYS arrive simultaneously. This is the null result.
  - Dot A position: if racePhase < 0.5, dot travels from cx toward mirror_A; if racePhase >= 0.5, dot returns from mirror_A to cx. Apply rotation transform.
  - Dot B: same logic along B arm.
  - Both dots are always at the same racePhase → same fractional distance along their arm → arrive back at center together.
- Collision point label: when both dots are near center (racePhase near 0.0 or 1.0), flash "SIMULTANEOUS ARRIVAL" text at center.
- Label: "Rotation: " + rotationAngle.toFixed(0) + "°" at top. "Both beams always arrive together" at bottom.

**Mode 3 — Interferometer fringe pattern:**

Draw the interferometer schematic on the left half (x: 0 to W*0.55) and the fringe pattern on the right half (x: W*0.55 to W).

Left schematic:
- Simple diagram: beam source at left, beam splitter at center, two arms (horizontal arm L₁ = L_base * armRatio, vertical arm L₂ = L_base). Mirror symbols at ends. Recombination at beam splitter → screen on right.
- Draw arm length indicators: L₁ label with current value, L₂ label with fixed value.

Right fringe pattern:
- Path length difference: `pathDiff = 2 * (L1 - L2)` where L1 = armRatio * L_base_px, L2 = L_base_px, L_base_px = 100.
- lambda = 40 (pixels, representing light wavelength — for fringe spacing).
- For each pixel row `py` from 0 to H: compute `intensity = 0.5 * (1 + Math.cos(2 * Math.PI * pathDiff / lambda + py * 0.1))`. The `py * 0.1` term adds a vertical fringe pattern.
- Map intensity to grayscale: `var g = Math.round(intensity * 220); ctx.fillStyle = 'rgb(' + g + ',' + g + ',' + g + ')'; ctx.fillRect(W*0.55, py, W*0.45, 1);`
- When armRatio = 1.0: pathDiff = 0, intensity = 0.5 * (1 + cos(py*0.1)) — uniform central bright fringe pattern. The null result is visually evident as equal-width fringes without a dominant dark center.
- Label: "Arm ratio: " + armRatio.toFixed(2) at top-right. "Equal arms: null result" label shown when Math.abs(armRatio - 1.0) < 0.05.
