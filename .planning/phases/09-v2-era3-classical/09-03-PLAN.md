## Objective

Implement two full interactive Canvas 2D simulations: (1) a Faraday induction sim where a draggable magnet induces a galvanometer deflection driven by the flux time-derivative (demonstrating Lenz's law reversal), and (2) a Carnot heat engine sim with a properly drawn PV diagram using isothermal hyperbolas and adiabatic power-law curves. ## Context

Both stops are `isStub: true`. Reference: stop-014 sim.js (ES5 IIFE pattern).

**Critical physics constraint for Stop 019:** The galvanometer needle must be driven by `dΦ/dt` — the time-derivative of flux — NOT by flux magnitude. Store `prevFlux` each frame. `inducedCurrent = (currentFlux - prevFlux) / dt`. The needle angle = `clamp(inducedCurrent * scale, -maxAngle, maxAngle)`. This makes the needle reverse direction when the magnet reverses (Lenz's law). See RESEARCH.md Pitfall 10.

**Critical physics constraint for Stop 020 (Carnot PV diagram pitfall):** Draw each segment between analytically-defined endpoints. Close the path explicitly with `ctx.closePath()` before `ctx.fill()`. Never rely on arc segments to accidentally close. See RESEARCH.md Pitfall 5.

**ES5 constraint:** All `var`, named functions, string concatenation. No arrow functions, no const/let, no template literals.

Era color: `rgba(82,133,200,0.8)` for canvas drawing.

Write a complete ES5 IIFE sim.

**Physics model:**
- T_H: hot reservoir temperature in Kelvin, controlled by slider (range 400–800 K, default 600 K).
- T_C: cold reservoir temperature in Kelvin, controlled by slider (range 200–400 K, default 300 K). Enforce T_C < T_H.
- Efficiency: `eta = 1 - T_C / T_H`. Display as percentage.
- Carnot cycle has 4 stages: (1) Isothermal expansion at T_H from V1 to V2, (2) Adiabatic expansion from V2 to V3 (T drops from T_H to T_C), (3) Isothermal compression at T_C from V3 to V4, (4) Adiabatic compression from V4 to V1 (T rises from T_C to T_H).
- Use n=1 mol of ideal gas. R=8.314. gamma=1.4 (diatomic).
- Define V1=1.0 (normalized). V2 = V1 * exp(W12/nRT_H) where W12 is the expansion work — but for display purposes, just set V2 = V1 * 2.0 (simplified ratio for visualization). Then for adiabatic: T_H * V2^(gamma-1) = T_C * V3^(gamma-1), so V3 = V2 * (T_H/T_C)^(1/(gamma-1)). V4 from adiabatic compression: T_C * V4^(gamma-1) = T_H * V1^(gamma-1), so V4 = V1 * (T_H/T_C)^(1/(gamma-1)).
- Scale V values to canvas pixel coordinates: V_max = max of V1..V4, mapped to graph right edge.

**Canvas layout (split: left = PV diagram, right = piston animation):**
- `splitX = W * 0.60` computed inside `resize()`.
- **Left panel — PV diagram:**
  - Graph axes: x = Volume (V1 to V_max * 1.1), y = Pressure (P_min to P_max * 1.1). P = nRT/V for isotherms.
  - Stage 1 (isothermal expansion at T_H): draw point-by-point path from (V1, P(V1, T_H)) to (V2, P(V2, T_H)). P = nRT_H/V. Color: red `rgba(220,80,80,0.9)`.
  - Stage 2 (adiabatic expansion): from (V2, T_H) to (V3, T_C). P = K/V^gamma where K = P(V2,T_H)*V2^gamma. Color: orange `rgba(200,140,50,0.9)`.
  - Stage 3 (isothermal compression at T_C): from (V3, P(V3, T_C)) to (V4, P(V4, T_C)). Color: blue `rgba(82,133,200,0.9)`.
  - Stage 4 (adiabatic compression): from (V4, T_C) to (V1, T_H). Color: teal `rgba(50,180,150,0.9)`.
  - Draw all four as `ctx.beginPath()` paths, each 30 point-by-point steps. Close all four into one filled path for the work area. Fill color: `rgba(82,133,200,0.12)` (translucent blue fill = work done).
  - IMPORTANT: after drawing fill, call `ctx.closePath()` explicitly. Never rely on the segments to accidentally close.
  - Animated point: a white dot traces the cycle continuously when running. `cyclePhase` advances from 0.0 to 1.0 then loops. Map phase to position on the four segments.
  - Labels: "T_H" (red) near stage 1 curve, "T_C" (blue) near stage 3 curve. Corner labels: V1..V4 on x-axis. Efficiency label: "η = " + (eta*100).toFixed(1) + "%" at top-right.
- **Right panel — piston animation:**
  - Simple piston: a rectangle (cylinder) with a movable piston head. Piston position = inversely proportional to current pressure. As cycle phase advances, piston moves left/right showing compression/expansion.
  - Hot source (red block) below cylinder in stages 1–2, cold sink (blue block) above cylinder in stages 3–4. These are visual indicators, not simulated heat flow.

**Controls:**
1. `hot-temp-slider` (range 400–800, step 10, default 600): updates T_H (in K). If T_H <= T_C, clamp T_H = T_C + 50.
2. `cold-temp-slider` (range 200–400, step 10, default 300): updates T_C. If T_C >= T_H, clamp T_C = T_H - 50.

**SimAPI:** start/pause/reset/destroy. Reset: T_H=600, T_C=300, cyclePhase=0.
